# Contributing Guidelines

Cảm ơn bạn đã quan tâm và mong muốn đóng góp cho dự án  
**Hệ thống Thương mại Điện tử – Mô hình phân tán dữ liệu (SQL Server & MongoDB)**.

Dự án được xây dựng chủ yếu cho mục đích **học tập, nghiên cứu và phát triển hệ thống thông tin**.

---

## 1. Nguyên tắc chung
- Mọi đóng góp đều được hoan nghênh dưới các hình thức:
    - Báo lỗi (bug report)
    - Đề xuất cải tiến (feature request)
    - Bổ sung tài liệu
    - Cải thiện mã nguồn
- Dự án sử dụng **MIT License**, mọi đóng góp phải tuân thủ giấy phép này.

---

## 2. Cách đóng góp
### Bước 1: Fork repository
Fork repository này về tài khoản GitHub cá nhân của bạn.

### Bước 2: Tạo nhánh mới
```bash
git checkout -b feature/ten-chuc-nang
```
### Bước 3: Thực hiện thay đổi

Tuân thủ cấu trúc project hiện có

Viết code rõ ràng, có chú thích khi cần thiết

Không commit các file nhạy cảm (mật khẩu, cấu hình riêng)

### Bước 4: Commit thay đổi
```bash
git commit -m "Mô tả ngắn gọn nội dung thay đổi"
```

### Bước 5: Tạo Pull Request
Gửi Pull Request về nhánh main kèm mô tả chi tiết nội dung đóng góp.

## 3. Quy ước mã nguồn

Tuân thủ chuẩn Java / Spring Boot (nếu áp dụng)

Đặt tên biến, hàm rõ nghĩa

Không commit file build hoặc thư viện dư thừa

## 4. Báo lỗi và đề xuất

- Sử dụng mục Issues trên GitHub

- Mô tả rõ:

    - Môi trường chạy

    - Các bước tái hiện lỗi

    - Kết quả mong đợi và kết quả thực tế
## 5. Ghi nhận đóng góp
Mọi đóng góp hợp lệ sẽ được ghi nhận trong lịch sử commit của dự án.
